Key behaviors
вАҐ Schema versioning: ServiceType.schemaVersion starts at 1 and auto-
increments when fields change. Providers store schemaVersionAtWrite
from the service type at write/activation time.
вАҐ Status transitions: Data-entry users can only save draft. Admin-
only transitions: draft->active|inactive, active->inactive,
inactive->active. Activating validates against the current schema and
refreshes schemaVersionAtWrite.
вАҐ Dynamic validation: Unknown fields rejected; type/enum/min/max/regex
enforced; forbids keys starting with $ or __proto__/constructor/prototype
at any depth.
вАҐ Public browsing: Only status=active providers are returned and only
fields with isPublic=true.
вАҐ Localization: String fields can be { en, ar } objects; admin responses
return bilingual fields unless ?langOnly=true.
